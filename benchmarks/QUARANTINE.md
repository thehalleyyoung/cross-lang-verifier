# Quarantined simulated benchmarks — DO NOT PUBLISH NUMBERS FROM THESE

Per `100_STEPS.md` Step 1 ("Delete or quarantine all simulated benchmarks"),
the following scripts contain **simulated / mock** result generation. Numbers
produced by them are **not** evidence and must never appear in the README,
papers, or any trusted results artifact.

The trusted, *real* results pipeline lives in `experiments/ub_divergence/`
(regenerated by `make reproduce`, confirmed against real compilers by
`make reproduce-confirm`). The CI guard `scripts/check_no_simulated_results.sh`
fails the build if:

* any committed `*.json`/`*.csv` results file contains a simulation marker, or
* a new `.py` under `benchmarks/`/`experiments/` introduces a simulation marker
  without being listed below.

## Quarantined files

- `benchmarks/sota_benchmark.py`
- `benchmarks/sota_benchmark_simple.py`
- `benchmarks/pairs/real_world_benchmark_pairs.py`
- `benchmarks/pairs/memory_benchmark_pairs.py`
- `experiments/ub_invisible_benchmark.py`
- `experiments/real_clib_experiment.py`
- `experiments/ablation_study.py`
- `experiments/pbt_baseline_experiment.py`
- `experiments/benchmarking/run_scaled_repair_experiment.py`

## Exit path

Each quarantined script should eventually be either (a) deleted, or (b) rewritten
to drive the real ground-truth re-execution harness
(`src/ub_oracle/reexec.py`) so its numbers become reproducible evidence.
