# CAPABILITIES — what this oracle soundly decides (and what it does not)

_Per `100_STEPS.md` Step 4. This is an honest, calibrated statement of scope. It
is intended to be the paper's threats-to-validity section in miniature. When in
doubt, the tool **abstains loudly** rather than claiming equivalence._

## Soundness direction

The oracle is **sound for divergence**: when it reports a *confirmed* divergence,
that divergence is real — it is exhibited by a **fully-defined input** on which a
real source build and a real target build observably differ (or on which the
source executes undefined behavior that a real optimizer exploits, while the
target is defined). It is **not** a sound equivalence prover: "no divergence
found" means only that none was found within the current search bound and the
supported divergence classes.

| Abstract-interpretation pre-pass | an interval-domain analysis (`abstract_interp.py`) runs **before** the per-class Z3 search and *proves* a class's UB region unreachable under the unit's declared operating range (`x_range`/`a_range`/`b_range`/`shift_range`), discharging it with **no solver call**. | CEGAR refinement (path-sensitive) | for guarded fragments the non-relational interval pre-pass cannot discharge, a **lazy predicate-abstraction CEGAR loop** (`cegar.py`) decides `∃x. UB(x) ∧ ⋀ assume_i(x)`: it begins with the UB condition and **no** guards (a sound over-approximation) and, on each spurious counterexample, **learns exactly the violated `assume`**, so it runs at most one solver call per guard. It returns `EQUIVALENT` only when even the over-approximation is UNSAT (UB unreachable under the guards), and a `DIVERGENT` verdict carries a concrete witness that satisfies every guard. Soundness is checked against exact enumeration of the (tiny) UB region, refinement is *measured* (solver calls / learned predicates / trace), and divergent witnesses are confirmed against the real C `-O0`/`-O2` builds via the existing signed-overflow harness | `src/ub_oracle/cegar.py` |
| k-induction (loops beyond bounded unrolling) | a textbook **k-induction** engine (`kinduction.py`) over a small integer transition-system IR lifts no-divergence claims past bounded unrolling. The **base case** starts from `init`, so any model is a *reachable* counterexample — a concrete loop-trip count at which the loop first goes undefined; the **inductive step** (optionally strengthened with auxiliary invariants the engine first *proves* inductive via `init ⟹ lemma` and `trans`-preservation, so strengthening is never unsound) certifies safety for an **unbounded** iteration count. Verdicts and witness depths are cross-checked against an independent concrete simulator; against real compilers the C loop traps under UBSan at *exactly* the predicted trip count (and not one trip earlier), and the SAFE verdict transfers across C→Rust/Go/Swift (identical defined output at a large trip count) | `src/ub_oracle/kinduction.py` |
| ABI / interop-divergence at FFI boundaries | a structural **ABI layout** checker (`abi_layout.py`) covering **structs, unions, enums and nested aggregates**: it computes the *exact* C-ABI layout (size, alignment, per-field offsets under the LP64 rules x86-64 SysV and AArch64 share for the modelled scalars) and flags an **interop hazard iff** a padding-optimizing representation would reorder a struct's fields, **or iff** a C `enum` and a default-repr Rust *fieldless* enum of the same arity have different widths (C `enum` = `int`/4 bytes; default Rust = the smallest integer that indexes the variants, e.g. 1 byte ≤256 variants). Soundness is anchored to real compilers: `clang` `sizeof`/`offsetof`/`_Alignof` confirms the C struct/union/enum/nested layouts; real `rustc` confirms `#[repr(C)]` reproduces them exactly while the default `repr(Rust)` **genuinely diverges precisely when** predicted (the observed divergent struct layout matches the optimized model byte-for-byte, and the default enum is exactly the predicted narrower width); Go's declaration-order layout matches C, so the hazard is specific to reordering/auto-sized representations and is never fabricated when the declared form is already canonical | `src/ub_oracle/abi_layout.py` |
| Byte-addressed provenance memory model (spatial/temporal safety) | a small **byte-addressed memory model** (`memory_model.py`) that decides spatial (out-of-bounds) and temporal (use-after-free, double-free) safety on whole allocation/free/load/store **traces**. Every pointer carries the **provenance** of its allocation, so an access is in bounds *iff* it lies in `[0,size)` of that object — never an adjacent allocation that happens to abut it — and any access through a freed object is a use-after-free. The model is not asserted but **confirmed**: each trace is emitted as C, compiled under **AddressSanitizer**, and ASan is checked to trap *iff* the model predicts a fault, reporting the **same fault kind** (heap-buffer-overflow / heap-use-after-free / double-free); model-safe and boundary (last-legal-byte) traces run clean | `src/ub_oracle/memory_model.py` |
| Structural cross-unit function alignment | a **function aligner** (`unit_alignment.py`) that matches each C function to its translated counterpart so divergence oracles compare the *right* pairs even under heavy renaming. Matching is decided by **structure, not names**: a signature score (matching arity plus C→target type families — `int`↔`i32`, `char*`↔`*const u8`, …, with an arity mismatch a **hard veto**) dominates, reinforced by a **self-reinforcing call-graph** score (once two functions are fixed, their callers/callees gain evidence), with name edit-distance only as a tiebreak. `align` is a deterministic greedy global assignment that honours user **pins**, leaves sub-threshold functions **unmatched** rather than forcing them, and on a module engineered to be adversarial to name matching (a 2-arg `add` whose name is closest to a 1-arg `add_one`) recovers the ground-truth pairing **exactly** while the `name_only_align` baseline does not | `src/ub_oracle/unit_alignment.py` |
| Soundness-frontier detection (loud abstention at foreign effects) | a **foreign-effect detector** (`foreign_effects.py`) that keeps the tool honest about the boundary of its pure-fragment semantics. A comment/string-aware lexical scanner flags every construct that leaves the analysable fragment — `volatile` accesses (observable side effects / MMIO), inline `__asm__` (opaque blob), calls to undefined `extern` functions (true FFI with an unknown body), atomics (cross-thread ordering contract), `setjmp`/`longjmp` (non-local control flow) and signal handlers (asynchronous interruption) — and `decide` returns a **loud ABSTAIN** with a human-readable reason naming each site, refusing to issue a divergence verdict rather than guessing; ordinary libc-only code (and language keywords) stay **CLEAR**. Crucially the abstention is *justified against real `clang` IR*, not assumed: a `volatile` read in a 4-iteration loop keeps **four** separate `load volatile` where the non-volatile version coalesces to **one** (so a value-folding pure model is provably unsound), inline asm appears as an opaque `call ... asm`, an `extern` callee is an undefined `declare`, and an `_Atomic` access lowers to `load atomic`. The per-pair frontier (what each target language uses to express each effect) is documented in `FOREIGN_FRONTIER` | `src/ub_oracle/foreign_effects.py` |
| Concurrency / data-race awareness (cross-language) | a **data-race oracle** (`concurrency.py`) that makes the C→{Rust,Go} concurrency migration story a *checked* fact rather than a claim. A catalogue of patterns — an unsynchronized shared counter, a mutex-guarded counter, a lock-free atomic counter, and read-only sharing — is compiled and **run under real race detectors**: the C side under **ThreadSanitizer**, the Go side under **`go run -race`**. The headline result is that the *same* unsynchronized-counter idiom is flagged as a data race on **both** the C source and the Go target, whereas Rust refuses to compile it at all (a shared `&mut` cannot cross a thread boundary without `Sync` — i.e. it must become `Mutex`/`Atomic`), and every synchronized variant runs **clean** under both detectors. A pattern is only ever labelled a race when a real sanitizer actually fires on a real binary (`confirm_race` requires the available detectors to agree with the prediction), and the per-target treatment is documented in `RACE_FRONTIER` | `src/ub_oracle/concurrency.py` |
| Function-pointer / indirect-call resolution | an **indirect-call resolver** (`indirect_calls.py`) for the dominant real-world idiom — a typed function-pointer **dispatch table** (syscall tables, plugin registries, hand-rolled vtables). It parses real C, and for a call `table[k](...)` computes a **precise** points-to set (exactly the functions named in the table's initializer) plus a **conservative, signature-typed** fallback (every defined function whose signature matches the table element type — and *no* function of a different signature, so a wrong-signature decoy and `main` are excluded). `table_is_well_typed` checks the precise set refines the conservative one. The resolution is **proven exact against real execution**: `confirm_table_dispatch` instruments every function with a stderr marker, compiles and runs the binary (whose `main` drives the table over every index), and verifies the set of functions actually reached through the table equals the predicted points-to set (`observed == predicted`, never escaping it) while the signature-incompatible decoy is never invoked. The per-target idiom (Rust `fn` pointers / `dyn Trait`, Go func values / interfaces) is documented in `INDIRECT_FRONTIER` | `src/ub_oracle/indirect_calls.py` |
| Real C preprocessing (macros / conditionals / includes) | a **preprocessing front door** (`preprocess.py`) that runs analysis only after the *real* C preprocessor (`clang -E`), because a C program's meaning is fixed only post-preprocessing. It proves three load-bearing facts against compiled, executed code: (1) macros are **semantically load-bearing** — `#define MUL(a,b) a*b` invoked as `MUL(1+1,2)` expands to `1+1*2` and a real binary evaluates it to **3**, whereas the parenthesized `((a)*(b))` gives **4**; `detect_unparenthesized_macros` flags the hazardous form up front and leaves the safe form, object-like macros and single-token wrappers alone; (2) `#ifdef`/`#if` conditionals **select the program** — the same source runs to 0 without `-DFEATURE` and 1 with it; (3) `#include` **resolution** makes a header-only symbol present after preprocessing and the built program links and runs. An analyser that skips preprocessing would compare a program that does not exist | `src/ub_oracle/preprocess.py` |
| Behavior-accurate libc / runtime modeling | executable, **behavior-accurate specs** for the most-hit libc surface (`libc_model.py`): `strlen`, `strcmp`, `strncmp`, `memcmp`, `memcpy`, `memset`, `strchr`, each a pure model over `bytes`/`int` that encodes the runtime's exact value contract — `strcmp`/`memcmp` return only the **sign** of the first differing unsigned byte; `strlen`/`strcmp`/`strchr` require NUL termination (the model raises on violation, surfacing the UB precondition); `memcpy` forbids overlap while `memmove` permits it. Every spec is **proven against the real libc** by randomized differential testing: `confirm_spec` builds a C harness that calls the host libc, feeds hundreds of random inputs (buffers passed as hex on argv), and compares real output to the model with **zero** mismatches. The `LibcSpec` + `confirm_spec` framework is runtime-agnostic — a new function is just a model plus a harness mode — and the UB/value contracts are documented in `LIBC_CONTRACTS` | `src/ub_oracle/libc_model.py` |
| High-fidelity ingestion from compiler IRs | facts are recovered from the **compilers' own** intermediate representations instead of hand-rolled parsing (`ir_ingest.py`): clang's `-ast-dump=json` yields exact function signatures, parameter names and storage classes (`add(int,int)->int`, `static char *dup_first(const char *)`), and rustc's `--emit=mir` yields **ownership facts for free** — a by-value non-`Copy` parameter (`Vec<i32>`) shows a `move`/`drop` of its MIR local and is recorded as *consumed*, whereas a `Copy` `i32` is not. Builtins are filtered by source-location provenance, both ingesters lower into a shared `IRModule`, and `confirm_clang_ingest`/`confirm_mir_ingest` re-derive and check these facts against the real clang and rustc on every run | `src/ub_oracle/ir_ingest.py` |
| Whole-project ingestion (build trees) | ingestion scales from a single hand-picked file to a **whole build tree** on both sides (`project_ingest.py`): the source side reads a Clang `compile_commands.json` compilation database (the CMake/Bear de-facto standard), recovers each translation unit's `-I` include directories, lowers every TU through the clang-AST ingester and unions their symbols into one `ProjectModule`; the target side enumerates a **Cargo workspace** via `cargo metadata` — the build graph Cargo itself uses — discovering every member package, target name, kind and source root. Both are proven against the **real** clang/cargo on generated multi-file projects (`confirm_compile_db` recovers `add`/`helper`/`slen` across two files with correct types and storage; `confirm_cargo_workspace` enumerates a two-member workspace with exact source roots) | `src/ub_oracle/project_ingest.py` |
| Solver portfolio + parallelism | decision procedures run as a **portfolio** rather than a single point of failure (`solver_portfolio.py`): z3 (in-process via the Python bindings) and boolector (out-of-process on a temp SMT-LIB2 file, exit 10/20 = sat/unsat) **race the same query in parallel** under a shared wall-clock timeout; the first decisive (`sat`/`unsat`) answer wins and **all answering solvers are cross-checked for agreement** — a genuine disagreement yields a loud `UNKNOWN`, never a silently-chosen verdict. A robustness battery spanning divergence-relevant bit-vector classes (signed overflow, unsigned wrap, truncation, shift-by-width, even-product low bit, xor-self) is solved by **every available solver** and matched against the known ground truth, with at least one query decided by **both** backends so the cross-check is real | `src/ub_oracle/solver_portfolio.py` |
| Fuzzer-guided frontend hardening | the C frontend is hardened by a **differential fuzzer** that uses the real clang as oracle (`frontend_fuzz.py`): a seeded generator emits random but **always-compilable, well-typed** translation units (random function counts, return types across the integer/pointer/void zoo, random arity, parameter types and storage classes); each program is compiled by clang (`-fsyntax-only`), ingested, and the recovered `IRFunction` set is **diffed against the generator's own ground truth** — names, arity, return types, parameter types and storage class must all match exactly. Dozens of programs survive each fixed-seed run with **zero parse-divergences and zero crashes**, and a corpus of malformed/non-C inputs (empty, truncated, unbalanced, NUL bytes, pathological repetition) makes the frontend return `None` rather than raise | `src/ub_oracle/frontend_fuzz.py` |
| Per-language conformance suite (merge gate) | a curated **regression corpus** pairing real constructs with their exact expected lowerings, checked against the real compilers (`conformance.py`). The C cases pin clang-AST facts — a `typedef`'d return type is preserved, an array parameter `int a[10]` **decays to `int *`**, a function-pointer parameter reconstructs to `int (*)(int)`, a `const int *` keeps its qualifier, `unsigned` canonicalises to `unsigned int`, and `static` linkage is read from the AST. The Rust cases pin rustc-MIR **ownership** — a by-value `Vec`/`Box`/`String` is *moved* while a reference or `Copy` scalar is not. The whole applicable suite is green (`confirm_conformance`), so any regression in how a construct lowers turns a case red — the merge gate every supported language passes | `src/ub_oracle/conformance.py` |
| Evaluation-order / sequencing oracle | C's evaluation-order rules are treated as a cross-language soundness hazard (`eval_order.py`). **Unsequenced modification** (`i = i++ + i++`, `g(i++, i++)`, `i = i++ + 1`) is genuine **undefined behavior** — the whole program's value is undefined — so it is detected precisely with clang's `-Wunsequenced` (the increment, call-argument and self-assignment idioms each yield a located diagnostic while clean sequenced code yields none) and answered with a **loud `ABSTAIN`**, because no target translation that picks a concrete order can be proven equivalent to another legal C compilation. **Unspecified argument-evaluation order with side effects** (defined left-to-right in Rust/Go but unspecified in C, and invisible in any single run) is documented as the sequencing soundness frontier | `src/ub_oracle/eval_order.py` |
| Known-bug / CVE corpus (proven end-to-end) | a curated **"we catch real bugs"** table of UB-rooted weakness classes, each tagged with its **CWE** and verified by actually compiling and running both sides (`cve_corpus.py`). Every entry pairs a C program that executes undefined behavior on a concrete input (confirmed by the **UBSan/bounds build trapping**) with a faithful Rust/Go translation that is fully **defined and deterministic**, and the re-execution harness confirms the definedness divergence. Covered: division by zero (CWE-369), out-of-bounds array read (CWE-125), signed integer overflow (CWE-190), shift past bit-width (CWE-758) and INT_MIN/-1 (CWE-682), spanning **C→Rust and C→Go** — every catch executed, not asserted | `src/ub_oracle/cve_corpus.py` |
| Labeled ground-truth set (sanitizer-established) | a **>=500-pair** labeled corpus across **two** language pairs (C→Rust, C→Go) whose `divergent`/`equivalent` label is fixed by **bounded enumeration + real sanitizers** — the C side is compiled under UBSan/bounds and run, and the label follows from the *trap* (divergent, with a defined deterministic target) or the *observable-output comparison* (equivalent), entirely **independent of the oracle** (`ground_truth.py`). This is the substrate that makes a precision/recall number defensible: on a full toolchain the sanitizer-established label agrees with the constructed label on every sampled item across both languages and both labels | `src/ub_oracle/ground_truth.py` |
| Scale-measurement harness (content-hashed results JSON) | drives the labeled corpus through the real decision procedure, recording **time/memory/verdict/abstention per item**, and emits a **canonical results JSON** (`scale_measure.py`). The *verdict layer* is content-hashed with sorted keys and stable separators while wall-clock/RSS live in a separate, explicitly **non-hashed** measurements section — so two independent runs on the same toolchain reproduce the **identical `content_hash`** even though their timings differ. ### What the executable oracles soundly establish

The **signed-overflow** oracle, for a unit `f(x) = x <op> c` over a C signed
integer type:

1. uses Z3 to find an input `x` for which `x <op> c` signed-overflows (UB), and
2. confirms, by compiling and running real binaries, that on that `x`:
   - the UBSan build **traps** (the UB is genuinely reachable),
   - the `-O0` and `-O2` builds **disagree** (the UB is *consequential*: the
     optimizer exploits it), and
   - the Rust translation produces a single **defined** value.

The **integer-model** oracles (shift-out-of-range, division/remainder by zero,
`INT_MIN / -1`) instead witness a **definedness divergence**: Z3 finds a defined
input on which

   - the UBSan build **traps** (C is undefined), while
   - the Rust translation is **defined and deterministic** — either a value or a
     clean, repeatable panic (confirmed by running Rust twice via the
     `trap_vs_defined` harness mode).

A divergence is reported as **confirmed** only when these conditions hold.

The **strict-aliasing** oracle uses a third mode, `optimizer_exploited`: no
sanitizer can trap a strict-aliasing violation, so the evidence is that the
**same C source produces different output at `-O0` vs `-O2 -fstrict-aliasing`** —
two builds of one deterministic program disagreeing proves the C result is
under-determined — while the Rust translation is a single defined value.

The **floating-point contraction** oracle (`fp_contraction`) reuses the
`optimizer_exploited` mode with a contraction-sensitive flag pair: C's `a*b + c`
may be fused into a single-rounding FMA at the implementation's discretion (C17
6.5p8 / `FP_CONTRACT`, *unspecified*), so `-ffp-contract=off` (two roundings)
and `-ffp-contract=fast` (one rounding) **disagree on the same source and input**,
while Rust's `a*b + c` always rounds twice (a single deterministic value). Z3's
floating-point theory finds a heavily-cancelling witness `(a,b,c)` where the
fused and unfused results actually differ in printed output.

## What it does NOT do yet (abstains / out of scope here)

- **Remaining divergence classes** (uninitialized reads, use-after-free,
  null-deref, eval-order, ...) are **catalogued but not yet executable oracles**.
  The framework (`src/ub_oracle/plugin.py`) is designed for them to be added as
  plugins; until then the tool **abstains loudly** (`NOT_COVERED`) rather than
  guessing.
- **Further language pairs** (Python→Rust, C++→Rust, ...) are part of the
  roadmap; **C → Go** is implemented and validated end-to-end (Step 37) as the
  generality proof that a new target only needs target-source emission plus a
  definedness predicate — the witness search and verifier are reused unchanged.
- **General whole-program equivalence**: no claim. The oracle reasons about the
  specific divergence classes it implements, on the units it is given.
- **Loops/recursion of unbounded depth, pointers/heap, concurrency**: not modeled
  by the current oracles.
- **Soundness for equivalence**: never claimed.

## Installation

The flagship verifier is a real, pip-installable distribution
(`cross-lang-verifier`) whose only runtime dependency is `z3-solver`; the
ground-truth confirmation step additionally discovers real compilers
(`clang`/`rustc`/`go`/`swiftc`) on `PATH` at run time.

```sh
pip install .                 # from a checkout (PEP 621 / pyproject.toml)
# or, from a built wheel:
pip wheel . --no-deps -w dist && pip install dist/cross_lang_verifier-*.whl
```

This installs the self-contained `ub_oracle` top-level package and two console
scripts — `cross-lang-verify` and the alias `cross-lang-verifier` — both bound
to `ub_oracle.cli:main`:

```sh
cross-lang-verify --units examples/units_manifest.json            # ground-truth
cross-lang-verify --units examples/units_manifest.json --no-confirm  # symbolic
```

`make package-check` is the reproducible proof: it builds the wheel, installs it
into a **fresh** virtualenv (with `PYTHONPATH` unset and run from a neutral
directory so nothing leaks in from the source tree), imports the installed
top-level package, and runs the console script end-to-end — confirming real
divergences against actual compilers and asserting the CI gate exit code.

## Reproducibility

- `make reproduce` regenerates `experiments/ub_divergence/results.json`
  deterministically (no toolchain, seeded randomness).
- `make reproduce-check` asserts byte-identical regeneration.
- `make reproduce-confirm` additionally writes `confirmations.json` from real
  compiler runs (environment-dependent; not part of the byte-identical check).
- `make guard` enforces that no simulated numbers enter trusted artifacts.
