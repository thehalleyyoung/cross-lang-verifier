# Adversarial near-miss corpus

Step 165 adds a small, persistent adversarial corpus that complements the broad
red-team grid and the 1,000-item negative corpus.  Each case is hand-crafted at a
decision boundary: a divergent control that must remain a `CANDIDATE` without a
toolchain, a safe twin one edit away that must be discharged by the interval
pre-pass as `NO_DIVERGENCE_FOUND`, or an unsupported/mis-probed unit that must
abstain as `NOT_COVERED`.

The checked manifest lives at
`experiments/adversarial_corpus/adversarial_corpus.json` and is regenerated by:

```bash
python3 -m experiments.adversarial_corpus.run --check
python3 -m pytest tests/test_adversarial_corpus.py -q
```

The committed manifest uses the deterministic no-toolchain path only; optional
live compiler replay separately proves divergent controls never fall to the
dangerous `NO_DIVERGENCE_FOUND`/`NOT_COVERED` verdicts.
